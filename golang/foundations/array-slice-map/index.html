Go Collections: Arrays, Slices and Maps

Learning Objectives

Concepts

  • Creating and using
    • Arrays
    • Slices
    • Maps
  • Using range to loop over arrays, slices and maps

Arrays

  • Have fixed size
  • collection of elements of the same type.
  • Elements can be any type

Array Syntax

[size]type or [size as int]type{comma separated literals}

Examples:

  • Example 1: [6]string string array with a size of 6
  • Example 2: [5]int{1,2,3,4,5} int array with a size of 5, and default values.
  • Example 3: []int{1,2,3,4,5} The same as example 2, except the size will be determined based on the number of initial elements. In this case, 5.

Other Characteristics of Arrays:

  • Arrays can be declared with the same rules as any variable.
  • An array’s element type cannot change
  • An array’s size cannot change

Array Declaration:

An example of declaring an array

dogs := [4]string{}
  • The above will create an empty array, that can hold up to 4 strings and assign it to the variable dogs.

Indexes

  • An index simply means the position a value is in.
  • The first element in an array is always at index 0
  • increments by one for each item in the array.

To visualize this, imagine the following table is a representation of our dogs array:

Index: 0 1 2 3
Elements: "” "” "” "”

Assigning Values

Assigning Specific Values
  1. Provide the index to the variable
  2. Provide the value to the right of = as with any other variable

Assigning Values by Index

dogs[0] = "German Shepard"  // First position,index 0
dogs[3] = "Boxer"   // Last position, index 3

This would change our table example to:

Index: 0 1 2 3
Elements: “German Shepard” "” "” “Boxer”
Initializing Arrays With Values

The dogs Array Initialized With Literals

dogs := [4]string{"Collie", "Labrador", "German Shepard", "Dalmatian"}
Index: 0 1 2 3
Elements: “Collie” “Labrador” “German Shepard” “Dalmatian”

Reading Values

To read a value from an array you need to reference the index of the element you want to read the value of.

Simple Read and Print:

dogs := [4]string{"Collie", "Labrador", "German Shepard", "Dalmatian"}
fmt.Println(dogs[2]) // Would print German Shepard

Variable Assignment Use Case:

Using the value of an array to declare a new string

dogs := [4]string{"Collie", "Labrador", "German Shepard", "Dalmatian"}
dog := dogs[2]
Determining a Collection

Use the built-in len() function you need to determine the size of:

  • Array

  • Slice

  • Map

  • The return value of len() is an int

  • See each section for examples of using len() with the specific collection.

Getting the length of our dogs Array:

dogs := [4]string{"Collie", "Labrador", "German Shepard", "Dalmatian"}
size := len(dogs)
  • In this example, size is assigned an int value of 4.
Quick Quiz

Given the following array:

numbers := [4]int{1, 2}
  • What type are the elements of this array?
  • Without using len(), what size is the size of the numbers array?
  • What is the value of the element at numbers[3]?

Slices

Slices are:

  • Look and get defined similarly to Arrays
  • Can grow in size
  • One of the types that get passed by reference by default

Basic Slice Declaration

dogs := []string{"Collie", "Labrador", "German Shepard", "Dalmatian"}
  • A slice is not defined with the number of elements as part of its type
  • The initial size will be the size of elements provide in the {}

Reading and Writing to Slices:

dogs[0] = "Boxer"
pet := dogs[1]
  • Changes the value of the element at index 0 to "Boxer"
  • Assigns the value Labrador to the new pet variable

Creating a slice from an Array or Another Slice

To create a slice from an array you can use the [low:high] expression:

What low and high mean
  • low: the lowest index you want to start at. In this case, index 1.
  • High: the highest index you want to stop at, but not include.
  • You can think of the high index as the value at index high - 1
  • High can also be the length of the array.

Using the [low:high] expression

arr := [5]string{"a","b","c","d","e"}
x := arr[1:3]
  • x now is a slice with [“b”,“c”]

How it works

Lets use the following table to explore how that works:

Index 0 1 2 3 4
Elements a b c d e
low:hi 1 3
Return b c

Methods to copy an entire array or slice:

arr := [5]string{"a", "b", "c", "d", "e"}

verbose := arr[0:len(arr)]

exact := arr[0:5]

omitlow := arr[:5]

omithi := arr[0:]

omitboth := arr[:]

Try Me!

  1. verbose: Using the starting index of 0 as low and the length of the array as high
  2. exact: Using the starting index of 0 as low and the literal 5 as high
  3. omitlow: Omitting low and the literal 5 as high
  4. omithi: Using the starting index of 0 as low and omitting high
  5. omitboth: Omitting both high and low

Sizing empty slices: Introducing make()

Why?
mySlice := []string{}
  • Empty slice
  • Has no zero value
  • Forced to resize right away ( see copy and append below)
  • Attempting to access would cause a fatal error

Creating a sized empty slice

mySlice := make([]string,10)

This creates and empty slice, with 10 string type elements.

Challenge
Given the String variable "challenge"
Then the numbers "3456" should be extracted as a string
And printed to the console

Hint:

  • A string is a slice of bytes
  • When accessing it, the value will be a uint
  • A byte can cast back to a string by using string(byte)

Try Me!

Slice Functions

You can grow a slice using one of the two built-in functions:

  • append
  • copy
append()

The append() built-in will allow you to grow a slice by adding elements:

Syntax

name = append(originalSlice, value1, value2...)

Append Examples

package main

import (
	"fmt"
)

func main() {
	slice1 := []int{1, 2, 3}

	// Append 1 or more individual elements to the slice
	slice1 = append(slice1, 4, 5)

	fmt.Println(slice1)

	slice2 := []int{6, 7, 8, 9}

	//Append combine 2 slices
	slice1 = append(slice1, slice2...)

	fmt.Println(slice1)

	slice3 := []int{5, 2, 3, 10, 11, 12, 0}

	//Select specific elements in a slice to append
	slice1 = append(slice1, slice3[3:6]...)

	fmt.Println(slice1)
}

Try Me!

  • ... is the spread operator and allows you to “spread” the values of a slice into into a variadic function

Append Best Practices:

  • The result of append should be assigned to the same variable name
  • If a new variable is needed with a larger size, use copy

As this example demonstrates, you can get unexpected results from appending to a new slice. This goes into detail on why

Using copy()

Use copy when you want to create new variable with the contents of the original slice

Copy to a larger slice

package main

import (
	"fmt"
)

func main() {
	slice1 := []int{1, 2, 3}
	slice2 := make([]int, 6)

	copy(slice2, slice1)

	fmt.Println(slice2)

}

Try Me!

Copy to a Smaller Slice
  • You can copy a larger slice into a smaller slice
  • Values that do not fit get dropped

Dropping elements as a result of a copy to a smaller slice:

package main

import (
	"fmt"
)

func main() {
	slice1 := []int{1, 2, 3, 4, 5, 6}
	slice2 := make([]int, 2)

	copy(slice2, slice1)

	fmt.Println(slice2)
}

Try Me!

Maps

  • Key/Value Collection
  • Keys can be defined as any comparable type
  • Keys are unique identifiers
  • value can be any type
  • Maps are a pass by reference type
  • Maps do not need to be sized

Using Maps

Syntax for declaring a map:

map[<key_type>]<value_type>
  • The <key_type> inside if the [] will determine the type of the key used to access the value
  • The <value_type> following the ] will determine the the type of the value for that element.

Creating Maps

Using Make

Initialize and use empty map

m := make(map[string]float64)   // Declare `m` to be a `map` that has a key type
                                // of `string` and a value type of `float64`.
m["somekey"] = 1.2              // Create keys using literal strings, and assign values
m["anotherkey"] = 3.4
Always Initialize Your Maps

Go will allow you to compile if your map is nil.

This will cause a runtime (nil pointer) error:

var m map[string]int

m["somekey"] = 1
Initialize a Map With Values

Creating a Map with Initial Values:

m := map[string]int{
		"a": 1,
		"b": 2,
	}
	fmt.Println(m["a"])
  • You’ll notice that there is a trailing comma on the final element.
  • This is required even if there is not another key/value pair.
  • You do not have to define the key/values on separate lines

Reading from map

You can read from a map like you would read from an array except that you use the key where you’d specify the index in the array.

m := make(map[string]float64))
m["somekey"] = 1.2
m["anotherkey"] = 3.4

d := m["anotherkey"]
fmt.Println(m["somekey"])

Accessing a map with a key returns 2 values:

  • The value found at the key, or the zero value of the defined type if the key is not found.
  • A bool value representing weather or not the key exists or not
Determining valid Keys

Supply 2 variables when accessing the map will allow you to check if the key is valid or not:

v, ok := mymap["key"]
Use Case:

You check the bool value because:

  • The value will always be the zero value of the type
  • The zero value may be a “good” value
// if the key is there, print the value
if v, ok := mymap["key"]; ok {
	fmt.Println(v)
}

Deleting a key

Syntax:

delete(map,key)

Example of deleting a key

package main

import (
	"fmt"
)

func main() {
	m := map[string]float64{
		"somekey":    1.2,
		"anotherkey": 3.4,
	}

	delete(m, "anotherkey")

	if _, ok := m["anotherkey"]; !ok {
		fmt.Println("The key was deleted")
	}
}

Range and Looping

You can loop over the contents of an iterable:

  • Using a traditional for loop based on the length of the iterable
  • Using the keyword range to automatically access the index/key and values of the iterable and automatically terminate the loop

Traditional Looping:

package main

import (
	"fmt"
)

func main() {
	array := [5]int{1, 2, 3, 4, 5}
	slice := array[:]

	fmt.Printf("\nArray Loop\n")
	for i := 0; i < len(array); i++ {
		fmt.Println(array[i])
	}

	fmt.Printf("\nSlice Loop\n")
	for i := 0; i < len(slice); i++ {
		fmt.Println(slice[i])
	}

	fmt.Printf("\nReverse\n")
	for i := len(slice) - 1; i >= 0; i-- {
		fmt.Println(slice[i])
	}
}

Range

Looping over iterables with range

package main

import (
	"fmt"
)

func main() {

	m := map[string]float64{
		"somekey":    1.2,
		"anotherkey": 3.4,
	}

	fmt.Printf("\nRange with map\n")
	for key, value := range m {
		fmt.Println(value, key)
	}

	slice := []int{1, 2, 3, 4}
	fmt.Printf("\nRange with map/slice\n")

	for index, value := range slice {
		fmt.Println(value, index)
	}
}

When range is used to to iterate over a string:

  • you get a rune value rather than a byte.
  • allow you to handle special characters in the string

Example:

package main

import (
	"fmt"
)

func main() {
	x := "Hello, "
	for i, v := range x {
		fmt.Println()
		fmt.Println("Rune:", v, string(v))
		fmt.Println("Byte:", x[i], string(v))
	}
}
